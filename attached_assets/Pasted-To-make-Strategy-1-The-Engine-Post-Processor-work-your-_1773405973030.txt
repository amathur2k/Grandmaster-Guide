Material Imbalance (More than just points)Standard points (P=1, N=3...) are too simple. To get better analysis, you should compute the Kaufman Imbalance.How to compute: Use python-chess to count pieces, then apply weights.The Logic: A Bishop pair is worth an extra +0.5. A Rook is worth more if the opponent has a minor piece.Pythonimport chess

def get_material_info(board):
    # Standard values
    weights = {chess.PAWN: 1, chess.KNIGHT: 3.25, chess.BISHOP: 3.33, chess.ROOK: 5.1, chess.QUEEN: 9.4}
    
    white_score = sum(len(board.pieces(pt, chess.WHITE)) * val for pt, val in weights.items())
    black_score = sum(len(board.pieces(pt, chess.BLACK)) * val for pt, val in weights.items())
    
    # Bonus for Bishop Pair
    if len(board.pieces(chess.BISHOP, chess.WHITE)) >= 2: white_score += 0.5
    if len(board.pieces(chess.BISHOP, chess.BLACK)) >= 2: black_score += 0.5
    
    return {"balance": white_score - black_score, "description": f"White is up {abs(white_score-black_score):.1f} in value."}

Piece Activity & MobilityThe LLM needs to know if a piece is "trapped" or "dominating."How to compute: Calculate the number of legal moves available to each piece.The Feature: If a Knight on the rim (h1) has 2 moves while a centralized Knight (d4) has 8, the LLM should know the latter is "highly active."Pythondef get_activity_metrics(board):
    metrics = {}
    for square in chess.SQUARES:
        piece = board.piece_at(square)
        if piece:
            # Count how many squares this specific piece attacks/can move to
            mobility = len(board.attacks(square))
            metrics[chess.square_name(square)] = {"piece": piece.symbol(), "mobility": mobility}
    return metrics
